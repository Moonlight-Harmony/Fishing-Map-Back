# 낚시기록지도

> 낚시 기록을 위치 기반으로 남기고, 어종별로 검색해 다른 사람들의 조과 포인트와 기록을 지도에서 확인하는 애플리케이션.

---

## 1. 문제 & 동기

### 어떤 문제를 풀고 싶은가?

- 낚시 기록은 사진첩, 메모장, 카카오톡, SNS 등에 흩어져 관리되기 쉽다.
- "어디서 어떤 어종이 잡혔는지"는 낚시에서 중요한 정보지만, 기존 기록 방식은 위치와 기록을 함께 보기 어렵다.
- 낚시 포인트 정보는 커뮤니티에 흩어져 있고, 실제 조과 기록과 연결되어 있지 않은 경우가 많다.
- 사용자는 공개하고 싶은 기록과 나만 보고 싶은 기록을 구분해서 관리하고 싶다.

### 왜 지금 이 프로젝트인가?

> 낚시 기록은 사진, 위치, 날짜, 어종이 자연스럽게 결합되는 데이터다.  
> 지도 기반으로 보여주면 개인 기록 앱이면서 동시에 커뮤니티형 정보 서비스로 확장할 수 있다.

## 3. 핵심 컨셉

낚시기록지도는 "낚시 일기"와 "지도 기반 조과 검색"을 합친 앱이다.

사용자는 낚시 기록을 남길 때 위치, 어종, 사진, 일시, 코멘트를 함께 저장한다. 다른 사용자는 지도에서 어종을 검색해 해당 어종이 잡힌 기록을 확인할 수 있다. 회원은 공개 기록뿐 아니라 자신이 남긴 비공개 기록만 따로 모아볼 수 있다.

핵심 경험:

- "최근에 감성돔 어디서 잡혔지?"
- "내가 작년에 우럭 잡았던 포인트가 어디였지?"
- "이 위치에서 다른 사람들은 어떤 어종을 잡았지?"
- "공개는 부담스럽지만 내 낚시 기록은 지도에 남겨두고 싶다."

## 6. 사용 흐름 초안

### 기록 등록 흐름

1. 사용자가 앱을 열고 기록 추가 버튼을 누른다.
2. 현재 위치를 사용하거나 지도에서 위치를 직접 선택한다.
3. 어종, 사진, 일시, 코멘트를 입력한다.
4. 공개 여부를 선택한다.
5. 저장하면 지도 위에 마커가 생성된다.

### 어종 검색 흐름

1. 사용자가 지도 화면에서 어종을 검색한다.
2. 해당 어종이 잡힌 공개 기록들이 지도에 마커로 표시된다.
3. 사용자가 마커를 클릭한다.
4. 앱이 사진, 잡힌 일시, 코멘트, 대략적인 위치 정보를 보여준다.

### 내 기록 보기 흐름

1. 회원 사용자가 로그인한다.
2. "내 기록만 보기" 필터를 켠다.
3. 공개 / 비공개와 관계없이 내가 저장한 기록만 지도에 표시된다.
4. 마커를 눌러 과거 기록을 다시 확인하거나 수정한다.

## 10. 주요 고민거리

### 위치 공개 문제

낚시 포인트는 민감한 정보일 수 있다. 정확한 위치를 공개하면 사용자가 기록 공유를 꺼릴 가능성이 있다.

초기 대응안:

- 공개 기록은 정확한 좌표 대신 반경을 둔 대략적인 위치로 표시
- 비공개 기록은 정확한 좌표 저장
- 사용자가 공개 범위를 직접 선택

### 기록 신뢰도

사용자가 임의로 기록을 올릴 수 있기 때문에 조과 정보의 신뢰도가 문제가 될 수 있다.

초기 대응안:

- MVP에서는 검증 기능을 넣지 않는다.
- 사진, 날짜, 코멘트 중심으로 자연스럽게 신뢰도를 판단하게 한다.
- 나중에 신고, 인증 뱃지, 활동 이력 등을 검토한다.

### 어종 데이터

어종명을 자유 입력으로 받을지, 정해진 목록에서 선택하게 할지 결정이 필요하다.

초기 대응안:

- MVP는 자유 입력 + 최근 입력 자동완성으로 시작
- 사용량이 늘면 표준 어종 목록을 만든다.

### 앱의 방향성

- 낚시 커뮤니티보다 기록 관리에 먼저 집중한다.
- 포인트를 빼앗긴다는 느낌보다 "내 기록이 쌓인다"는 느낌을 우선한다.
- 공개 기록은 다른 사람에게 도움을 주되, 작성자의 위치 노출 부담을 줄인다.
- 지도 화면이 앱의 중심 경험이 되어야 한다.
